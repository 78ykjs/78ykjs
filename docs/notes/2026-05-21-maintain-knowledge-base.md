# 如何维护这个知识库

## 核心目标

这个知识库的目标不是堆积资料，而是把反复遇到的问题、真正理解的知识、做项目得到的经验沉淀下来。

## 维护节奏

- 每天：把零散想法放入 `inbox`。
- 每周：整理 1 到 3 条内容为正式笔记。
- 每个项目结束后：写一篇项目复盘。
- 每个月：检查目录结构和索引是否还好用。

## 判断一条内容是否值得沉淀

- 以后可能再次用到。
- 解释了一个之前不清楚的问题。
- 来自一次真实项目或真实错误。
- 能总结为步骤、清单、经验或模板。

## 最小可用格式

```markdown
# 标题

## 问题

## 结论

## 过程

## 参考
```
