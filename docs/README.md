# 个人知识库

这是一个面向长期积累的个人知识库。它采用轻量目录结构，适合记录学习、项目、论文、工具、问题解决过程和阶段性复盘。

## 快速入口

- [Inbox](inbox/README.md)：先记录，再整理。
- [Notes](notes/README.md)：沉淀后的知识卡片。
- [Projects](projects/README.md)：项目过程、成果、问题和复盘。
- [Areas](areas/README.md)：长期关注领域和能力建设。
- [Resources](resources/README.md)：外部资料、论文、书籍、链接和工具。
- [Templates](templates/README.md)：常用写作模板。

## 推荐工作流

1. 捕捉：任何想法、资料、问题先进入 `inbox`。
2. 整理：每周挑选有价值的内容，改写为 `notes`。
3. 连接：为笔记补充相关项目、资源和主题领域链接。
4. 复盘：项目完成后写总结，把经验沉淀到领域笔记。
5. 清理：长期无价值的临时内容可以删除或归档。

## 命名建议

文件名建议使用：

```text
YYYY-MM-DD-topic.md
```

示例：

```text
2026-05-21-github-knowledge-base.md
```

## 内容原则

- 一篇笔记只解决一个核心问题。
- 保留来源和上下文，方便以后追溯。
- 先写清楚，再追求完整。
- 能复用的经验要沉淀为模板、清单或步骤。
