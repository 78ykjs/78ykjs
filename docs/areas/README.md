# Areas

这里放长期维护的主题领域。领域不是一次性完成的项目，而是需要持续积累的能力和方向。

## 领域索引

| 领域 | 说明 |
| --- | --- |
| software-engineering | 软件工程、项目结构、代码质量、协作流程 |
| automation | 脚本、自动化工作流、效率工具 |
| research-writing | 论文阅读、资料整理、技术写作 |

## 领域页建议结构

```markdown
# 领域名称

## 当前水平

## 关键问题

## 学习路线

## 相关笔记

## 相关项目

## 资源清单
```

## 维护方式

- 每个领域可以有一个总览页。
- 把相关笔记、项目和资源链接回来。
- 定期更新当前问题、学习路线和阶段性成果。
