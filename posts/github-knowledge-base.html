用 GitHub Pages 搭建个人知识库

3 minute read · Updated: May 21, 2026

这个仓库的目标是同时承担个人主页和公开知识库两个角色。主页负责展示入口，`docs` 目录负责保存长期内容。

目录结构

知识库采用轻量分区：`inbox` 放临时记录，`notes` 放整理后的知识卡片，`projects` 放项目记录，`areas` 放长期领域，`resources` 放外部资料，`templates` 放常用模板。

写作流程

  1. 先把想法放进 Inbox，不在第一步追求完美。
  2. 每周整理有价值的内容，改写成独立笔记。
  3. 把笔记关联到项目、领域或资源页。
  4. 项目结束后补充复盘，让经验能被下一次复用。
这个网站是静态 HTML 和 CSS，不依赖复杂构建流程。推送到 GitHub 后，只要 Pages 指向当前分支即可访问。

下一步

后续可以继续加入搜索、文章归档、代码高亮、图片画廊和自动生成索引。目前先保持简单，让内容能稳定积累。
